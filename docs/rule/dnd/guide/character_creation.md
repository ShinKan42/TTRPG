## **背景**

:::

综合 [团前准备](./preparation) 中的 [角色准备](./preparation#角色准备) 为自己的角色设计故事线路与个人特点等等

由于背景所提供的各类加成都会在下文中分类解决，此处 ==只需构想内心故事与角色弧光==

::: note 官方背景

可到 [官方背景](../data/origin/designated) 中匹配官方设定好的背景，但由于刻板、单调而极不推荐

请尽量想象 ==自己喜欢的、能调动自身感情去关心== 的角色背景

:::

> [官方背景](../data/origin/designated)通常有以下数据，已拆分到下方各步骤：
>
> - 属性：`(A)` 选择 2 项，分别 +2 & +1；`(B)` 选择 3 项，皆 +1。不论如何，不得超过 20
> - 专长：选择 1 项 [起源专长](../data/feat/origin) 获得熟练
> - 技能：选择 2 项 [技能](../data/skill) 获得熟练
> - 工具：选择 2 项 [工具](../data/tool) 获得熟练
> - 装备：`(A)` 获得 [官方背景](../data/origin/designated) 特定装备；`(B)` 50 GP； `(C)` 通过下文村规解决
> - 某种特殊的扮演小能力

::: note 版本区别

在 2024 （5R）版里，属性是位于背景中而且可自由选择，这与 5E 的跟随种族固定不同

:::

::: tip 信仰

信仰是中世纪奇幻，也是 DND 相当重要的一环，并不亚于魔法

不妨设想一下角色是否有什么信仰，==信仰的神明是哪一方面的==

不需要知道世界背景对应神的细节，甚至可以是现实世界、其他作品等来源的神明，重要的是 ==神明所代表的价值观==

:::

::: info 阵营

阵营是角色 ==三观与处事方式== 的快速总结，其存在着极大的模糊性

==阵营只是一种倾向，不是一种禁忌，也可以随着故事而改变==

大多数时候是 ==对扮演进行自我约束的参考==，但在少数情况也会有 ==游戏实际上的影响==

阵营没有好坏，只是对角色的一种反映、剧情矛盾的体现与玩家的自我约束，也是角色人生、信仰与死后的位面归属

以下是各阵营具有代表性的一句话介绍：


|     |  | **守序** | **中立** | **混乱** |
|:-------:|:-------:|:------:|:------:|:------:|
|  |         | `自有行为准则` |        | `万法有何不可` |
| **善良**  | `尊重生命与共情` | 正道就应如此 | 平凡好心是真 | 就说善不善吧 |
| **中立**  |         | 世界公平运转 | 尊重他人命运 | 怎么了我乐意 |
| **邪恶**  | `他人皆可垫脚石` | 不爽规则别玩 | 谁也别想挡我 | 唯恐天下不乱 |

`守序` 与 `混乱` 的矛盾在于：是否有 ==自我约束== 的行为准则，可以是自己设立的；如，“不杀人准则”与各种誓言

请注意，守序并不代表其准则就是公共良俗，混乱也并非见规矩就要破坏

`善良` 与 `邪恶` 的矛盾在于：世间/他人的 ==生命可否是自己目标的工具与手段==，是对他人生命是否尊重/共情/怜悯

请注意，善良并不代表圣母爱心泛滥，邪恶也能为自己利益而行善

最典型的反直觉例子是：守序善良的圣武士也可以是为心中正义心狠手辣、不择手段的狠角色

!!另有为无理智生物准备的：“无阵营”，一般不适用!!

:::

## **种族**

:::

结合角色的背景主题、人生命题、个人线路，以及自身的喜好等因素，选择[种族](../data/race/index)

::: note 自定种族

不要害怕，就算规则里没有，也不妨自己设定一个种族

最关键的是，设定一下这个种族的特殊能力、种族天赋/优势

:::

> [官方种族](../data/race/index)通常有以下数据：
>
> - 介绍（常见姓名、种族寿命、文化故事）
> - 生物类型（如：类人）
> - 体型（巨/大/中/小/微，一般为中型）
> - 速度（一般为 30ft / 6 格）
> - 种族特质（部分种族有不同分支可供选择）

::: info 混血种族

由于 2024 (5R) 的 `玩家手册` 未给出混血种族的信息，混血处理办法有下：

1. 根据向前兼容原则，==使用 5E 版本== 的混血种族
2. 使用某一单一种族，==假装是混血==
3. 申请并讨论，==挑选两种种族的部分特性缝合==
4. 申请并讨论，==原创 DIY== 一个新种族

:::

## **职业**

:::

根据角色的==战斗风格、特殊能力==，选择对应的 [职业](../data/class/index)

角色的等级代表的即是所有 `职业` 等级的总和

`职业` 在 3 级后，会出现细分的 `子职`

如果说 `职业` 代表了超能力的 ==印象概念与使用条件==，那么 `子职` 就代表了 ==细分差异与如何运用==

::: tip 方向建议

详细介绍可到 [职业](../data/class/index) 查看列表。但不妨快速思考下列方向：

- 有什么 ==特殊能力==？（如` 野蛮人` 的怒气狂暴、`德鲁伊` 的动物变形、`魔契师` 的宗主契约）
- 用什么 ==战斗方式==？（如 `战士` 的老练技艺、`游荡者` 的诡诈偷袭、`吟游诗人` 的演奏激励）
- ==力量来源== 于何方？（如 `术士` 的天生资质、`法师` 的后天学习、`牧师` 的信仰神恩）
- ==独特细分== 又如何？（近战还是远程、战斗还是功能、专注于什么方面）
- 联系上 ==角色扮演==？（筋肉人近战 `术士`？控制欲命令 `牧师`？江湖老骗子 `法师`？）

请注意，所谓的战斗输出与团队搭配并非必要，==体现角色故事与好玩== 才是唯一重要的

:::

::: warning 兼职

`职业` 的所有信息一般只针对本 `职业`，只有一般规则与通用资源共享

因此，尽管 `术士` 可以创造 `法术位` 使用 `圣武士` 的 `至圣斩`，但兼职 1 级 `法师` 并不能抄写其他职业得来的法术

同理，通过不同 `职业` 获得的同一个 `额外攻击`/`无甲防御` 特性是不叠加的，但 `疾风连击` 与 `偷袭` 却都不冲突

:::

## **属性**

:::

角色的 `属性` 反映了其不同方面的能力

物理三项：
- `力量(STR)`：蛮力、运动、跳高跳远，使用力气解决
- `敏捷(DEX)`：矫健灵敏、精准控制，运用身体的机巧
- `体质(CON)`：身体素质与健康，抵御疾病、醉酒等异常

精神三项：
- `智力(INT)`：逻辑思维、记忆能力，并非普世意义的智力，用于回忆推理，更类似于思考与谋略
- `感知(WIS)`：感官五感、精神坚韧、处世常识，常用于侦察与抵抗精神侵扰，更类似于灵性与慧根
- `魅力(CHA)`：社交手腕、气质谈吐，并非一定是外貌美丑，更多是政治技巧与吸引力，更类似于领袖才能

::: info 属性值与调整值

`属性值`：Ability Score，大数值，一般不直接使用，从 10 起步，代表凡人平均水平，每 +2 点为 1 档 +1

`调整值`：Ability Modifier，小数值，直接用于各方面，如 `伤害骰` = <Badge type="danger" text="武器伤害骰 + 属性调整值" />

`调整值` 的计算方式 = <Badge type="warning" text="（属性值 / 2） - 5 （向下取整）" />

如：8-9 对应 -1；10-11 对应 0；12-13 对应 +1，以此类推

:::

### **初始属性**

建卡加点的方式分为 3 种，请选择喜欢的方式：

::: tabs

@tab 固定数值

以下 6 个数值，自由分配

15、14、13、12、10、8

@tab:active Roll 点

重复以下方法 6 次，确认 6 个数值，自由分配：

<Badge type="danger" text="4d6 去除最低值后求和" />

因此，最高为： 6 + 6 + 6 + 任意

而最低为： 1 + 1 + 1 + 1

@tab 购点

使用 27 点购点费用，根据以下费用购点：

13 及以下：1 点费用

14-15：2 点费用

16 及以上：官方不支持，考虑使用 3 点费用

:::

### **起源属性**

即 `背景`/`种族` 属性，==任选以下一种==，对 `属性值` 进行操作：

1. 挑选 2 项分别：+2 与 +1
2. 挑选 3 项全部：+1

::: danger 注意

初始与起源的 `属性值` 叠加后，不得超过 20

:::

## **技能**

:::

[技能](../data/skill) 是角色通用的能力，因此也是最常用的机制，反映角色 ==做某类事务的能力==

1. 根据 `职业` 提供的 `技能` `熟练` 范围与数量，选择对应的 `熟练` `技能`

2. 参考 `背景` 经历设定，随意选择 2 项 `技能` 获得 `熟练`

::: info `熟练` 与 `专精`

`熟练加值`：Proficiency Bonus，根据角色等级得出的数值，反映角色对特定事物的 ==擅长程度==

`熟练`：Proficient，加 1 层 `熟练加值`

`专精`：Expertise，加 2 层 `熟练加值`；部分职业可提供，必须先有该 `技能` 的 `熟练`

:::

::: note 万事通

`吟游诗人` 在计算时（如不使用半自动卡），请务必注意不要漏掉 `万事通`（2级） 的特性

:::

## **法术**

:::

根据 `职业` 查阅 `施法` 特性，了解 [如何使用](../data/spell/index#施法规则)

::: note 名词解释

简要来说：

`戏法`：Cantrip，==无限次数== 使用的小 `法术`

可准备/已知 `法术`：根据 `职业` 相关规则，准备并更换 `法术` 使用，==没有准备/未知的 `法术` 无法被使用==

`法术位`：Spell Slot，`法术` 可以使用的 ==次数==

`法术环`：Spell Level，`法术` 根据 ==强度== 进行的分类

:::

## **数据**

:::

DM 会根据您的选择，处理、填入、记录绝大部分数据

::: note 防具

此时可优先决定穿戴的护甲/盾牌：

1. 请查阅 `职业`，确定 `熟练`，也即合适使用的护甲类型与盾牌

2. 从以下初始护甲中选择一项：

| **名称**  | **类型** | **护甲 `AC`** | **`敏捷` `AC`** | **`力量`** | **`隐匿`** | **穿戴(min)** | **脱下(min)** |
|:-------:|:------:|:-----------:|:-------------:|:-----------:|:-------------:|:-------------:|:-------------:|
| **皮甲**  | 轻甲     | +1          | 不限制           |             |               | 1             | 1             |
| **链甲衫** | 中甲     | +3          | 2             |             |               | 5             | 1             |
| **鳞甲**  | 中甲     | +4          | 2             |             | `劣势`          | 5             | 1             |
| **链甲**  | 重甲     | +6          | 0             | >=13          | `劣势`          | 10            | 1             |

3. 选择是否携带盾牌（AC +2，占用 1 只手，穿脱时间 = 🟢`标准动作`）

数据计算方法可参照 [常用骰法](./quick_roll)

:::

::: info 生命值

生命值计算的时刻到了，选择以下一种方式：

1. 根据 `职业` 对应的 `生命骰` 决定投骰大小并 Roll 点

2. 同上，但取 数学期望 +1 为结果，如 d8 对应 4 + 1 = 5

3. 1 级时，取 `生命骰` 最大数值

得出结果后，`最大生命值` 增加：结果 + `体质` `调整值`

:::

## **资源**

:::

请记录自身拥有的特性（来自 `种族`、`职业`、`专长`等），注意 ==次数与恢复方式==，方便 ==备忘== 与查阅

::: info `短休` / `长休`

休息为不间断的休整；

其中 `短休` 为 1 小时：可使用 `生命骰` 恢复 `生命值`：结果 + `体质` `调整值`；恢复对应资源

`长休` 为 8 小时：可恢复 `生命值`、角色等级一半数量的 `生命骰`、各种对应资源

:::

## **村规**

:::

详见 `村规` 栏目，影响建卡的主要如下：

- DIY 点数：建卡/个人线发展提供 DIY 点数，设计特殊增强
- 每日开支：开支内物品默认拥有
- 随身物品：以有限随身与背包区别，替换负重计算
- 衣食住行：增强扮演与资源管理
- `工具` 用法：`工具` 提供常规使用方法
- `长休` 活动：`长休` 需要分工与检定、规划有限行动以确保效果

另外有其他 `村规`，主要如下：

- `暴击` 增强：3R 规则沿用，`暴击` 判定与一击必杀
- BP 点数：每回合 1 点，兑换行动经济与易伤
- 黑白对骰：`大成功` / `大失败` 获得白/黑骰，主动增强检定
- 命运硬币：`长休` 1 枚/烧血获取，Yes / No 对应 +10 / -5
- 答案之牌：挂机、OB、无聊的补救措施，根据牌义修改剧情
- 塔罗激励：用塔罗牌替换 `英雄激励`，大幅增强效果
